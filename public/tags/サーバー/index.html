【ジャンク再生計画 Vol.4】「ECCメモリ」って何？ サーバー用メモリの安さと罠

CPU、GPUときて、今回は 「メモリ」 です。 もともとこのZ4G4にはmicron製のDDR4の8GBのメモリが２枚挿さっていました。 普通のパソコンなら「DDR4の8GBをもう何枚か買えばOK！」で終わる話ですが、相手はワークステーションでCPUはXeon。そう簡単にはいきません。 今回は、我々を悩ませる 「ECCメモリ」 と、その種類のについて解説します。 1. そもそも「ECCメモリ」とは？ 普通のPC用メモリ（Non-ECC）と違い、ECC（Error Correction Code） メモリには「データのエラーを自動で訂正する機能」がついています。 普通: なんらかのエラーでデータが1ビット化けた → ブルースクリーンで落ちる ECC: データが化けた → 自動で直して動き続ける 「24時間365日止まってはいけない」サーバーやワークステーションのための機能です。 この機能のおかげで、Xeonマシンは非常に安定しているわけですね。 このECC機能の影響で、動作が1クロック分遅くなるという弱点はありますが、大体2000Mhzを超える動作の1クロックは誤差の範囲です。 2. 「Registered」と「Unbuffered」の罠 ここからが本題です。 実はXeonで使えるECCメモリには、大きく分けて2種類あります。 ECC UDIMM (Unbuffered): 普通のメモリにECC機能がついただけのもの。一部のCore iシリーズや一部のRyzenでも使えたりする。 ECC RDIMM (Registered): 信号を整える「レジスタ」チップが載ったガチ業務用。 この2つ、形は同じDDR4なのに混ぜると動きません。 そしてマザーボードによって「どっちしか対応していない」という縛りがあったりします。 今回のHP Z4 G4は、公式には 「RDIMM」 が推奨されています。（UDIMMも動く場合がありますが、混在は不可） 3. なぜサーバー用メモリは安いのか？ ヤフオクやメルカリを見ると、中古なら普通のDDR4メモリより、サーバー用（RDIMM）の方がちょっと安いことに気づきます。 「8GBで1000円！？」みたいなことがたまにあります。 理由は単純でして、 データセンターの世代交代で大量に廃棄されるから、そして普通のPCには刺さらないからです。 需要と供給の関係で、我々のようなジャンクPC愛好家にとって、RDIMMはまさに宝の山なのです。 もちろん業務用なので、普通に買うとびっくりするぐらい高いです。 4. 今回の戦利品 というわけで、今回確保したメモリがこちら。 Sk hynix製 DDR4-2666V ECC RDIMM 8GB × 3枚 = 24GB ...
